Course Outline & Syllabus

Course Description

This laboratory module will accompany the course with the same name given by Prof. Roberta Gori and Prof. Andrea Corradini. The objective is to put into practice the theory presented in the course and in this laboratory. Students will implement interpreter and compiler stages for different, albeit simple, programming languages. At the end of the course students will have implemented both static-analysis techniques and translation to low-level intermediate representation, such as LLVM IR. The final grade will be determined both by a final project and an oral exam.

Syllabus

  • Interpretation of (simple) language semantics
  • Implementation of a type system for a functional programming language
  • Control-flow graph construction
  • Data-flow analysis and its usage in optimization passes
  • Compilation to LLVM IR

Project Assignment

At the end of the course the students are required to submit a project containing both code and a report. The project will be composed of multiple fragments. For each topic presented during the lesson, it will be required to implement the techniques described in the corresponding material. Some detail will intentionally be left unspecified. The students must be able to solve them on their own. The course does not mandate a specific programming language for the implementation. However, the students are required to discuss the language and tools they intend to use with the teacher beforehand. In the report they must explain the implementation choices and the issue encountered during the implementation. Furthermore, the report must contain precise instructions on how to run the submitted code.

This is an individual project. While you may discuss high-level concepts with classmates, all code must be your own original work. Use of AI tools must be documented. Any form of plagiarism will result in a zero for the assignment and may lead to further academic consequences.
